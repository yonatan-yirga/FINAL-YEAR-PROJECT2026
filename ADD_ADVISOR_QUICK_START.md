# Add Advisor Feature - Quick Start Guide

## 🎯 What's New?

Department Heads can now register new advisors directly from the system!

---

## 🚀 How to Use

### For Department Heads:

#### Option 1: From Advisors Page
1. Go to **Advisors** page
2. Click the green **"Add Advisor"** button (top right)
3. Fill in the form
4. Click **"Register Advisor"**

#### Option 2: From Dashboard
1. Go to **Department Dashboard**
2. Find **"Quick Navigation"** section
3. Click **"Add Advisor"**
4. Fill in the form
5. Click **"Register Advisor"**

---

## 📝 Form Fields

| Field | Required | Description | Example |
|-------|----------|-------------|---------|
| **Full Name** | ✅ Yes | Advisor's full name | Dr. John Doe |
| **Email** | ✅ Yes | Must be unique | advisor@university.edu |
| **Phone Number** | ✅ Yes | Contact number | +251 912 345 678 |
| **Staff ID** | ✅ Yes | Must be unique | STAFF-2024-001 |
| **Max Students** | ❌ No | Default: 15 | 15 |

---

## ✅ What Happens After Registration?

1. ✅ Advisor account is created
2. ✅ Login credentials are sent to advisor's email
3. ✅ Advisor can login immediately
4. ✅ Advisor appears in Advisors list
5. ✅ You can assign students to this advisor

---

## 📧 Email Sent to Advisor

The advisor receives an email with:
- Welcome message
- Login email
- Temporary password
- Login URL
- Recommendation to change password

## 🎯 Key Benefits

1. **No Self-Registration**: Advisors can't register themselves (more control)
2. **Instant Access**: Advisors can login immediately
3. **Email Notification**: Advisors receive credentials automatically
4. **Department Association**: Advisors automatically assigned to your department
5. **Workload Management**: Set max students per advisor

---

## 🐛 Troubleshooting

### "Email already exists"
- This email is already registered in the system
- Use a different email address

### "Staff ID already exists"
- This staff ID is already in use
- Use a different staff ID

### "Failed to register advisor"
- Check your internet connection
- Ensure all required fields are filled
- Contact system administrator if issue persists
